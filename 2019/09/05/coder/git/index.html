git 基础用法

发表于 2019-09-05 | 分类于 IT 日常 |

git 基础

git 三种状态

已提交(committed)、已修改(modified)、已暂存(staged)

git 三个工作区域

远程仓库、工作目录、暂存区域

  • 在工作目录修改文件 –> 已修改
  • 将文件快照放入暂存区域 –> 工作目录 to 暂存区域 –> 已暂存
  • 提交更新 –> 暂存区域 to 远程仓库 –> 已提交

git 配置

git 配置有三个作用域，其应用优先级：local > global > system。

  • --system 配置文件为 /etc/gitconfig 下，所有用户的配置
  • --global 配置文件为 ~/.gitconfig 或 ~/.config/git/config 下，只针对当前用户的配置
  • --local 配置文件为 proj/.git/config，只针对当前仓库的配置

查看配置

  • 查看所有配置项 git config [--system|--global|--local] --list
  • 查看特定配置项 git config [--system|--global|--local] user.name

如果不指定作用域，则按优先级获取，即在 local 取不到该项的值再从 global 取，还是取不到再从 system 取，如果 system 中也没配置，则返回空。

设置配置

如果不指定作用域，则默认为 --local。

删除配置项

如果不指定作用域，则默认为 --local。

查看帮助

git 仓库

创建远程仓库

git remote add <remote-name> <url>

查看远程仓库

git remote
git remote -v(显示url)
git remote show <remote-name>(查看某一远程仓库的详细信息)

重命名远程仓库

git remote rename <old-name> <new-name>

移除远程仓库

git remote rm <remote-name>

从远程仓库抓取

git fetch <remote-name>
此命令会抓取远程仓库的所有分支的引用，不会进行合并，必须手动合并分支
git pull <remote-name> <branch-name>
如果本地一个分支设置为跟踪一个远程分支，使用pull可自动抓取并合并远程分支到本地分支

推送到远程仓库

git push <remote-name> <branch-name>
当前远程仓库没人抢先推送时才能推送成功，否则必须先抓取远程分支合并后再推送。

Git分支

创建/获取Git仓库

创建本地工作目录：git init
克隆服务器仓库：git clone <url> <name>

跟踪文件

文件状态

未跟踪(untracked)、已修改(modified)、已暂存(staged)、未修改(unmodified)/已提交(commited)

检查文件状态

  • git status
  • git status -s
  • git status --short

跟踪新文件

git add <file>/<dir>

跟踪修改的文件

git add <file>/<dir>
每次修改文件后，提交更新前都要重新跟踪这个修改的文件，因为提交更新时是暂存区的版本，而不是本地目录的版本。

忽略文件

创建一个.gitignore文件来指定哪些文件不需要跟踪

  • *.[oa] 所有以.o或.a结尾的文件
  • _*~_ 所有以~结尾的文件
  • build/ build目录下的所有文件
  • /build 本目录下的build文件，不是目录，也不包含其它目录的build文件
  • doc/*.txt doc根目录下的所有文本文件
  • doc/*/.txt doc目录下的所有文本文件

查看已暂存/未暂存修改

  1. 查看未暂存文件的修改信息：
    git diff
  2. 查看已暂存文件的修改信息：
    git diff -cached
    git diff -staged

提交更新

  1. 打开编辑器：git commit
  2. 直接输入说明：git commit -m <statement>
  3. 跳过暂存区域直接提交：git commit -a

移除文件

  1. 从暂存区域和本地目录移除：git rm <file>
  2. 只从暂存区域移除，保留在本地目录成为未跟踪文件：git rm --cached <file>
  3. 如果文件修改过且放入暂存区域，删除时必须强制删除：git rm -f <file>

移动文件

git mv <oldfile> <newfile>
等价于下面三条命令集
mv <oldfile> <newfile>
git rm <oldfile>
git add <newfile>

查看提交日志

git log

  • -p 显示每次提交的内容差异
  • - 只显示最新的n次提交
  • –stat 显示简略的统计信息
  • –pretty 指定显示的格式

撤销操作

重新提交

git commit --amend
第二次提交会覆盖上一次的提交，如果第二次提交没有新的修改，就会使用第一次的快照，只覆盖提交信息

撤销暂存

git reset HEAD <file>

撤销修改

git checkout -- <file>
将文件修改为上次提交的样子或者刚放入工作目录时的样子，直接拿以前的文件来覆盖，因此很危险

分支简介

  • git分支本质上仅仅是指向提交对象的可变指针，提交对象提交时分支会自动向前移动。
  • git init会默认创建一个名为master的分支，这个分支和普通分支没什么区别。
  • HEAD才是特别的分支，它指向当前所在的本地分支，可看作当前分支的别名。切换分支时HEA D会指向另一个分支，同时把本地目录的文件替换为另一个分支指向的文件。

分支创建

git branch <branch-name>

分支切换

  • git checkout <branch-name>

  • git checkout -b <branch-name>

    会同时创建分支并切换到该分支，等价于下面两条命令集
    git branch <branch-name>
    git checkout <branch-name>

分支合并

git merge <other-branch>
将other-branch合并到当前分支，合并后other-branch的状态就是merged，可直接删除。

查看分支

  • 查看所有分支：git branch
  • 查看所有分支最后一次提交：git branch -v
  • 查看所有已合并分支：git branch --merged
  • 查看所有未合并分支：git branch --no-merged

删除分支

git branch -d <branch-name>
只能删除那些已经合并的分支，如果想强制删除未合并的分支，使用
git branch -D <branch-name>
