【Lua 浅入浅出】元表的秘密

发表于 2021-02-22 | 分类于 Lua |

在 Lua 中，可以为 table 或 userdata 定义元表来定义其行为，string 有一个共享的元表，无法更改但可以修改这个共享元表，其它类型没有元表也无法设置元表。元表实现了 table 的“运算符重载”，同时也是面向对象的根基。

什么是元表

  • Lua 中可以为数据类型定义一套预定义的操作集，表示这个类型的值可以进行什么操作，这个操作集就是它的元表；
  • 对一个变量值进行某种运算时，会首先查找它有没有元表，元表中有没有这个运算对应的方法，这个方法称之为元方法；
  • 有点类似于高级语言中的运算符重载，即给数据类型定义一个运算符映射的方法，当其对执行运算符操作时就会执行这个方法。

    举个例子，假设 a, b 是非数字类型的变量，执行 a+b 操作时，会查找 a 或 b 的元表中有没有 + 运算符对应的元方法，即 __add 方法，有则调用这个元方法，没有则操作不合法。

谁有元表

只有 table，string 和 userdata 类型有元表,

  1. 所有 string 变量共享一个元表且默认就有元表且无法更改元表，但可以修改这个共享元表，比如扩展元方法；
  2. 每个 table 变量可以设置一个共享元表也可以设置独有的元表，无默认元表；
  3. 其它类型的变量既无默认元表也无法设置元表。
  • 函数 getmetatable 用于获取一个值的元表，返回一个 table 或 nil；
  • 函数 setmetatable 给一个值设置一个元表，如果可以的话。

元方法

在元表中，很多运算符都有对应的元方法名，只要定义了对应的元方法，即可进行运算符操作。
